# Студенческий концепт UniverID

## 1. Краткое описание
Цель проекта — собрать в едином цифровом центре дневники учебы и кампуса: расписание, заметки по предметам, личный профиль студента, клубы и мероприятия, доступ через QR, а также возможности по карьере (вакансии, стажировки). Продукт рассчитан на студентов и преподавателей технологического института и призван повысить прозрачность учебного процесса и вовлеченность в кампус.

## 2. Исходная идея (по рисунку)
- Единый модульный интерфейс "кампус в одном приложении":
  - Расписание занятий по неделям с разбивкой по дням.
  - Заметки и конспекты по предметам (электронная заметка к каждому курсу).
  - Карта/профиль студента и "цифровая карта" кампуса.
  - Внеучебная жизнь: клубы, секции, мероприятия.
  - QR-доступ: пропуск через QR-код для прохода и взаимодействия с сервисами.
  - Раздел вакансий и стажировок (для карьерного роста).
  - Цифровой навигатор по университету (этажи, кампус, аудитории).
  - Секции и клубы по интересам.
  - Пакет пропусков: студенческий билет, пропуск в общежитие, транспортная карта, абонементы и скидки.
- Очередовательная архитектура: MVP с минимальным набором модулей, затем добавление функционала и интеграций.

## 3. MVP-область и ключевые модули
- Обязательные на первых парах (MVP):
  - Расписание занятий (Schedule)
  - Заметки по предметам / Электронная заметка (Notes)
  - QR-доступ для кампуса (QRAccess)
  - Цифровой навигатор (Campus Navigator)
  - Секции и клубы (Clubs & Sections)
- Расширяемые модули (для будущих спринтов):
  - Attendance (посещаемость)
  - Profile (профиль студента)
  - Careers (карьера)
  - Events & Clubs (мероприятия)
  - Student Services (обслуживание)
  - Passes (пакет пропусков: студбилет, транспорт, абонементы)
  - Admin/Faculty (админ)

## 4. Архитектура и стек
- Фронтенд: чистый JS/HTML/CSS или минимальная сборка; возможно переход к React/Vue в зависимости от команды.
- Архитектура: модульная, каждый модуль — отдельный компонент
- Данные (демо-данные):
  - student: идентификатор, имя, курс, кафедра, GPA, факультет
  - schedule: неделя/день/занятие/предмет/тип/аудитория/преподаватель
  - notes: по предметам, темы, содержимое
  - clubs/events: список клубов и мероприятий
  - vacancies: вакансии и стажировки
  - qr: структура для доступа (token, validUntil)
- API: на старте — демо-данные; далее — REST/GraphQL API для расписания, заметок, QR, вакансий и т.д.
- UI-экосистема: карточки/плитки, сетки, цветовые токены; поддержка адаптивности (desktop/mobile); темизация (светлая/темная тема)

## 5. Модели данных (скелет)
- Student: id, name, email, faculty, department, course, group, gpa, enrolledDate
- Schedule: week, day, time, subject, type, room, teacher
- Notes: subject, topic, content, lastUpdated
- Clubs/Activities: name, category, schedule, description, enrollment
- Events: name, date, time, location, type
- Vacancies: company, position, type, format, salary, deadline, description
- QRAccess: token, validUntil, userId
- CampusNavigator: building, floor, rooms, pointsOfInterest, description
- Passes: studentTicket, transportCard, buildingAccess, seasonTickets

## 6. Архитектурная раскладка по модулям
- Schedule: день недели, занятия, отступы
- Notes: заметки по предметам
- Attendance: (MVP) пропуски
- Profile: студент-данные
- Events & Clubs: афиша, секции, клубы
- Careers: вакансии
- QR Access: пропуски
- Campus Navigator: карта корпусов, этажи, аудитории, навигация
- Passes: студенческий билет, транспортная карта, пропуск в общежитие, сезонные абонементы

## 7. Цвета и дизайн
- Цветовая палитра (интерфейс):
  - Primary: синий
  - Secondary: голубой
  - Accent: зелёный
  - Background: темно/светлая тема
- Токены цветов: --primary, --bg-card, --text-muted, --contrast

- Accessibility: контраст, keyboard navigation

## 8. Пользовательские сценарии
- Главная: плитки модулей
